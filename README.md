# Система управления фитнес-центром (Часть 1)

Бэкенд-приложение (REST API) для автоматизации учета клиентов и тренеров фитнес-центра. Разработано в рамках учебной практики строго по техническому заданию.

## Обоснование технологического стека
*   **Язык программирования:** C# (.NET 8.0 / 9.0) — обеспечивает строгую типизацию, высокую производительность и безопасность работы с данными.
*   **Архитектурный подход:** ASP.NET Core Minimal APIs — современный, легковесный подход для построения REST API без лишнего шаблонного кода (boilerplate).
*   **Документация:** Swagger / OpenAPI (`Swashbuckle.AspNetCore`) — интегрирован для автоматической генерации интерактивной документации и тестирования эндпоинтов напрямую из браузера.

---

## Архитектурные слои приложения
Проект спроектирован с учетом разделения ответственности (Separation of Concerns). Это позволит безболезненно заменить in-memory хранилище на реальную базу данных (Entity Framework Core):

1.  **`FitnessPP.Models` & `Dtos`** — Чистые сущности данных (`Client`, `Trainer`) и объекты передачи данных для сложных ветвлений/вложенностей JSON.
2.  **`FitnessPP.Repositories`** — Слой абстракции данных. Использует интерфейсы (`IClientRepository`, `ITrainerRepository`) и их реализации со списками в оперативной памяти (`InMemory...`).
3.  **`FitnessPP.Services`** — Слой бизнес-логики (`FitnessService`). Отвечает за валидацию данных, обработку статусов и связывание сущностей (клиент ↔ тренер).
4.  **`Program.cs`** — Маршрутизация (Контроллеры/Эндпоинты) и конфигурация DI-контейнера.

---

## Инструкция по запуску
1. Откройте решение `FitnessPP.sln` в Visual Studio.
2. Установите зависимость Swagger (если не установлена автоматически):
   `Install-Package Swashbuckle.AspNetCore` через Консоль диспетчера пакетов.
3. Запустите проект, нажав **`Ctrl + F5`**.
4. Перейдите в браузере по адресу для тестирования: `https://localhost:XXXX/swagger/` (где ХХХХ — ваш локальный порт).

*   **Запрос:** `POST /api/trainers`
*   **Тело запроса:**
