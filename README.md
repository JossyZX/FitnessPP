# Система управления фитнес-центром (Часть 2)

Бэкенд-приложение (REST API) для автоматизации учета клиентов, тренеров, шкафчиков и дополнительных услуг фитнес-центра. В данной части осуществлен полный переход от in-memory хранилищ к полноценной реляционной базе данных.

## Обоснование технологического стека
*   **СУБД:** SQLite — реляционная база данных. Выбрана из-за компактности (хранится в одном файле `fitness.db` внутри проекта), что позволяет преподавателю запустить и проверить проект без развертывания тяжелых серверов (PostgreSQL/MySQL).
*   **ORM:** Entity Framework Core (EF Core) — стандарт индустрии для .NET. Используется для управления схемами, связями и автоматического выполнения SQL-запросов (JOIN) через LINQ.
*   **Миграции:** Настроено автоматическое применение миграций и развертывание схемы БД при каждом старте приложения (`context.Database.Migrate()`).

## Автоматическое наполнение БД (Seed-данные)
При самом первом запуске приложения база данных автоматически наполняется следующими системными данными:
1.  **20 шкафчиков** с номерами от 1 до 20 (изначально свободны, `ClientId = null`).
2.  **5 дополнительных услуг** согласно ТЗ:
    *   `SOLARIUM` — Солярий (400 руб.)
    *   `POOL` — Бассейн (200 руб.)
    *   `SAUNA` — Сауна (0 руб.)
    *   `CRYOSAUNA` — Криосауна (1000 руб.)
    *   `CROSSFIT` — Кроссфит (500 руб.)

## Реализованная бизнес-логика (Валидация по ТЗ 5.4)
1.  **Аренда шкафчика (1:1):** Шкафчик можно занять только в том случае, если он свободен (`ClientId == null`). При попытке занять чужой шкафчик сервер возвращает `HTTP 409 Conflict`.
2.  **Ограничение на количество шкафчиков:** У клиента не может быть двух шкафчиков одновременно. Повторный запрос вернет `HTTP 400 Bad Request`.
3.  **Покупка услуг (M:M):** Если клиент повторно пытается подключить уже купленную им услугу, запрос успешно обрабатывается (`HTTP 200 OK`), но дублирования записей в связующей таблице не происходит (запрос игнорируется).

## Инструкция по запуску и проверке
1. Откройте проект в Visual Studio.
2. Нажмите **`Ctrl + F5`** для запуска.
3. Перейдите по адресу `https://localhost:XXXX/swagger/` для открытия интерактивной документации.
4. Файл базы данных `fitness.db` будет автоматически создан в корневой папке проекта.
